▶️ AnyLog Demo – Quick Start

A concise, operator-friendly guide to launching, connecting to, and exploring the AnyLog edge data fabric demo.

OVA boots → containers auto‑start GUI, Query, Operator x2 Local‑only endpoints

Start AnyLog Edge Data Fabric/Historian

If the installation does not autostart, launch Terminal and run the following. This only needs to be done the first time the VM starts.

cd ~/AnyLog
./ALinstall.sh -d -s install

This starts:

  • gui-1 – Web UI for monitoring and data management
  • grafana – Sample Dashboard
  • AnyLog-Demo-standalone – Master / Operator / Query node
  • AnyLog-Demo-operator2 – Second data operator node

Note: After first start, wait several minutes while AnyLog services initialise.

AnyLog CLI

To connect to the AnyLog CLI on either node:

cd ~/AnyLog/node/docker-compose
  • make attach ANYLOG_TYPE=anylog-standalone – query & operator 1 CLI
  • make attach ANYLOG_TYPE=anylog-operator – operator 2 CLI

🤖 Claude Desktop – MCP Integration

Claude Desktop can connect directly to the AnyLog MCP server on this VM, letting you query industrial data, inspect policies, and run SQL using natural language.

1 · Install Claude Desktop

  1. Download Claude Desktop from https://claude.ai/download and install it on your host machine.
  2. Sign in with your Anthropic / Claude account.

2 · Locate the config file

OSConfig file path
macOS ~/Library/Application Support/Claude/claude_desktop_config.json
Windows%APPDATA%\Claude\claude_desktop_config.json
Linux ~/.config/Claude/claude_desktop_config.json

Create the file if it does not already exist.

3 · Add the AnyLog MCP server entry

Paste the block below into claude_desktop_config.json. If the file already has a mcpServers key, merge just the inner "anylog-demo" object into the existing map.

{
  "mcpServers": {
    "anylog-demo": {
      "type": "sse",
      "url": "http://vmipaddr:32149/mcp/sse"
    }
  }
}

4 · Restart Claude Desktop

Quit and reopen Claude Desktop. In Settings → MCP Servers you should see anylog-demo listed with a green connected indicator.

5 · Verify the connection

Open a new conversation and try:

List the databases available on the AnyLog network.

Claude will invoke the listDatabases MCP tool and return results inline.

Useful starter prompts

  • "What tables are in the new_company database?"
  • "Run SQL: SELECT * FROM rand_data LIMIT 10"
  • "Show the status of all AnyLog nodes."
  • "List all blockchain policies of type operator."
MCP endpoint (after install): http://vmipaddr:32149/mcp/sse
